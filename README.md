#Digital Booking — Full Stack Web Application

Proyecto integrador desarrollado durante la carrera Certified Tech Developer (Digital House).
Aplicación web de reservas construida con arquitectura full-stack y metodología ágil Scrum.

Tecnologías utilizadas:
Frontend:
.HTML5
.CSS3
.JavaScript
.React

Backend:
.Java
.Spring Boot
.Arquitectura REST API

Testing:
.Jest (unit testing)
.Postman
.Selenium

DevOps y metodología:
-Git / GitLab
-Scrum
-CI/CD pipelines

Funcionalidades principales:
.Registro y autenticación de usuarios
.Búsqueda y reserva de alojamientos
.Consumo de APIs REST
.Validación de formularios
.Testing funcional y documentación de defectos

Mi rol en el proyecto:
.Desarrollo de interfaces responsive en React
.Integración del frontend con APIs REST
.Validación de formularios y manejo de estados
.Testing y documentación de casos de prueba
.Trabajo colaborativo con metodología Scrum
.Cómo ejecutar el proyecto localmente

Backend:
1-Crear base de datos:
MySQL Workbench_CREATE DATABASE digitalbooking;
CREATE USER 'root'@'%' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON digitalbooking.* TO 'root'@'%';

2-Ejecutar la aplicación Spring Boot desde el IDE o con Maven.

Frontend
Bash
npm install
npm start
La app se ejecutará en:
http://localhost:3000

Testing
Se realizaron:
.Pruebas unitarias de componentes con Jest
.Testing exploratorio basado en flujos de usuario
.Documentación de casos de prueba y defectos por sprint

Qué aprendí con este proyecto:

.Trabajo en equipo en un entorno ágil real

.Desarrollo de aplicaciones full-stack

.Integración frontend-backend mediante APIs REST

.Flujo profesional de versionado con Git

.Importancia del testing en el ciclo de desarrollo
